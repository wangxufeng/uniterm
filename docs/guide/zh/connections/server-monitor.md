# 服务器监控

服务器监控功能通过 SSH 连接到远程 Linux 服务器，实时采集并展示系统运行指标，帮助运维人员快速掌握服务器状态。

> 注意：服务器监控仅支持 **Linux** 远程服务器，依赖 `/proc` 文件系统和 Linux 特有命令。需要 SSH 认证信息（密码或密钥）。

## 打开方式

在已建立的 SSH 连接上，可通过以下入口打开监控面板：

| 入口 | 操作 |
|------|------|
| 标签页右键菜单 | 右键 SSH 终端标签页 → **打开服务器监控** |
| 面板菜单 | 点击面板标题栏 `...` → **打开服务器监控** |
| 侧边栏右键 | 右键 SSH 连接 → **打开服务器监控** |

打开后，服务器监控以独立标签页的形式展示，与终端、文件传输等标签页并列，可拖拽排序。

## 性能

实时展示 CPU、内存、磁盘和网络四项核心指标，每 1 秒自动刷新，以 60 点折线图展示最近 1 分钟的历史趋势。

### CPU

- 总使用率百分比及折线图
- 核心数
- 进程总数和文件句柄数
- 1 分钟 / 5 分钟 / 15 分钟负载均值

### 内存

- 总量、已用、可用、使用率
- Cached 和 Buffers 占用

### 磁盘

- 根分区（`/`）总量、已用、使用率

### 网络

- 接收速率和发送速率（字节/秒），通过两次采样差值计算

## 进程

列出当前 CPU 占用最高的 **30 个进程**，每 1 秒自动刷新。

![进程列表](/imgs/monitor_light.png)

| 功能 | 说明 |
|------|------|
| 排序 | 默认按 CPU 降序排列 |
| 搜索 | 按进程名、用户名或 PID 过滤 |
| 暂停 | 点击暂停按钮停止刷新，再次点击恢复 |
| 进程详情 | 点击任意进程行，右侧滑出详情抽屉 |

### 进程详情

点击进程后，右侧弹出抽屉面板，展示以下信息：

- **基本信息** — PID、PPID、名称、状态、线程数、可执行文件路径、工作目录、完整命令行、启动时间
- **文件描述符** — 总数及分类统计（文件、socket、管道、匿名、设备）
- **虚拟内存** — VmRSS、VmSize、VmPeak、VmData、VmStk、VmExe、VmLib
- **CPU 与上下文切换** — CPU 时间片、自愿/非自愿上下文切换次数
- **I/O 统计** — 读/写字符数和字节数

### 发送信号

在进程详情底部，可选择信号类型向进程发送信号：

| 信号 | 编号 | 说明 |
|------|------|------|
| TERM | 15 | 正常终止（默认） |
| KILL | 9 | 强制终止 |
| HUP | 1 | 挂起，常用于重新加载配置 |
| INT | 2 | 中断，相当于 `Ctrl+C` |

发送前会弹出确认对话框，强制终止（KILL）有额外警告提示。

## 端口

手动刷新，列出所有正在监听的 TCP/UDP 端口。

| 列 | 说明 |
|------|------|
| 协议 | TCP / UDP |
| 本地地址 | 监听地址和端口 |
| 进程 | 进程 PID 和名称 |

通过 `ss -tulnp` 采集（失败时回退到 `netstat -tulnp`）。

## 磁盘

手动刷新，以树形结构列出所有块设备。

| 列 | 说明 |
|------|------|
| 名称 | 设备名，子分区缩进显示 |
| 类型 | disk / part / rom |
| 大小 | 设备总容量 |
| 挂载点 | 挂载路径 |
| 已用 / 使用率 | 已用空间及百分比 |
| 介质 | HDD / SSD / ROM |
| 文件系统 | ext4、xfs 等 |
| UUID | 设备 UUID |
| 厂商 / 型号 | 硬件信息 |

通过 `lsblk -J` 和 `df -h` 采集，介质类型根据 `rota` 标志判断（1 = HDD，0 = SSD）。

## 网卡

手动刷新，列出所有网络接口。

| 列 | 说明 |
|------|------|
| 名称 | 接口名（eth0、wlan0 等） |
| 状态 | UP / DOWN |
| MAC | MAC 地址 |
| 速率 | 接口速率（Mbps） |
| 类型 | Physical / Bridge / Bond / Virtual / Loopback |
| Bond 主接口 | 所属 Bond 接口（如有） |
| IP 地址 | 绑定的 IP 地址列表 |

通过 `ip -j link show` 和 `ip -j addr show` 采集，自动检测 Bond 和 Bridge 接口。

## 系统信息

连接建立时一次性采集，展示服务器的静态配置信息。

| 类别 | 内容 |
|------|------|
| 系统 | 操作系统（`/etc/os-release`）、版本、内核（`uname -r`）、主机名、时区 |
| 硬件 | CPU 型号、核心数、架构（`uname -m`）、CPU 频率、总内存、总磁盘 |
| 网络 | 本机 IP |
