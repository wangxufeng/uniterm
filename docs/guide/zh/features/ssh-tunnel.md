# SSH 隧道

uniTerm 内置 SSH 隧道管理器，支持三种转发模式，可创建、管理和监控多个隧道连接。

![SSH 隧道](/imgs/ssh_tunnel_light.png)

## 隧道管理器

SSH 隧道在侧边栏独立标签页中管理。点击左侧边栏的隧道图标打开隧道面板，查看所有已创建的隧道及其运行状态。

- **状态指示灯** — 绿色表示隧道正在运行，灰色表示已停止
- **右键菜单** — 右键隧道项可启动、停止、编辑或删除隧道
- **拖拽分组** — 拖拽隧道项到分组中，便于分类管理

## 三种转发模式

### 本地转发 (-L)

将远程服务器的端口映射到本地，通过本地端口访问远程服务。

**典型场景**：远程服务器的数据库（如 MySQL :3306）不对外开放，通过本地转发映射到 `localhost:3307` 后，本地工具即可直接连接。

```
本地端口 :3307  →  SSH 隧道  →  远程数据库 :3306
```

### 远程转发 (-R)

将本地端口暴露给远程服务器，远程主机可通过其本地端口访问你的本地服务。

**典型场景**：本地开发的服务需要让远程服务器或外部用户访问，如本地 Web 应用通过远程转发让公网服务器可访问。

```
远程端口 :8080  →  SSH 隧道  →  本地服务 :3000
```

### 动态转发 (-D, SOCKS5)

在本地搭建 SOCKS5 代理，浏览器或其他应用通过代理访问远程网络中的所有服务。

**典型场景**：通过跳板机访问公司内网的所有 Web 系统、API 等，无需逐个配置端口转发。

```
浏览器  →  SOCKS5 :1080  →  SSH 隧道  →  远程网络中的所有服务
```

## 创建隧道

1. 在隧道面板中点击 **新建隧道**
2. 选择 **SSH 连接**作为跳板机（需先在连接列表中创建 SSH 连接）
3. 选择 **转发模式**：本地转发、远程转发或动态转发
4. 根据模式填写本地地址/端口和目标地址/端口（动态模式仅需本地端口）
5. 可选：设置 **自动启动**，应用启动时自动连接该隧道

## 自动启动

开启自动启动后，隧道会在 uniTerm 启动时自动连接。适用于需要始终保持连接的隧道，如开发环境的 SOCKS5 代理。

## 代理链

多个隧道可以级联组合。一条隧道的目标可以指向另一条隧道的入口，实现多层网络穿透。
